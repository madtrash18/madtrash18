# get_next_line

> 42 Gyeongsan — Circle 1

파일 디스크립터(fd)로부터 개행 문자 단위로 한 줄씩 읽어오는 함수를 구현한 프로젝트입니다.  
`BUFFER_SIZE` 매크로로 읽기 단위를 조절할 수 있으며, 여러 fd를 동시에 처리하는 보너스도 구현했습니다.

---

## 함수 프로토타입

```c
char *get_next_line(int fd);
```

- 호출할 때마다 다음 한 줄을 반환 (`\n` 포함)
- 파일 끝에 도달하면 마지막 줄 반환 후 이후에는 `NULL` 반환
- 메모리 누수 없이 동작

---

## 핵심 구현 개념

```
[fd로부터 BUFFER_SIZE만큼 읽기]
        ↓
[정적 변수에 누적 저장]
        ↓
['\n' 발견까지 반복 읽기]
        ↓
['\n'까지 한 줄 추출 → 반환]
[남은 데이터는 다음 호출을 위해 정적 변수에 보존]
```

### 사용한 핵심 개념

| 개념 | 설명 |
|------|------|
| **정적 변수(static)** | 함수 호출 간 버퍼 상태를 유지 |
| **파일 디스크립터** | `read(2)` 시스템 콜로 fd 직접 읽기 |
| **동적 문자열 연결** | 버퍼를 반복적으로 이어붙이는 `ft_strjoin` 패턴 |

---

## 보너스

여러 fd를 동시에 처리할 수 있도록 정적 변수를 배열로 관리:
```c
static char *stash[OPEN_MAX];  // fd별 독립적인 버퍼 유지
```

---

## 빌드 / 사용

```bash
cc -Wall -Wextra -Werror -D BUFFER_SIZE=42 get_next_line.c get_next_line_utils.c -o gnl
```

---

## 배운 점

- `static` 변수의 생명주기와 다중 호출 간 상태 유지 메커니즘
- 파일 디스크립터와 `read()` 시스템 콜 이해
- 버퍼 크기가 성능에 미치는 트레이드오프 분석
- 메모리 누수 없이 누적 버퍼를 관리하는 설계 패턴

# philosophers

> 42 Gyeongsan — Circle 3

**식사하는 철학자 문제(Dining Philosophers Problem)** 를 POSIX 스레드로 구현한 프로젝트입니다.  
멀티스레딩, 뮤텍스, 타이밍 제어를 통해 데드락 없이 시뮬레이션을 구현합니다.

---

## 문제 정의

```
N명의 철학자가 원형 테이블에 앉아 있습니다.
각 철학자 사이에 포크가 1개씩 있습니다.

철학자는 세 가지 상태를 반복합니다:
  생각(thinking) → 포크 집기(eating) → 잠자기(sleeping)

식사하려면 왼쪽·오른쪽 포크 2개가 모두 필요합니다.
철학자가 정해진 시간(time_to_die) 안에 식사하지 못하면 사망합니다.
```

---

## 실행 형식

```bash
./philo [철학자 수] [time_to_die] [time_to_eat] [time_to_sleep] [선택: 최소 식사 횟수]

./philo 5 800 200 200
./philo 5 800 200 200 7   # 각 철학자가 7번 먹으면 종료
```

---

## 구현 방식

### 동시성 문제 해결

| 문제 | 해결책 |
|------|--------|
| **데드락** | 짝수 번호 철학자는 오른쪽 포크를 먼저 집도록 순서 조정 |
| **레이스 컨디션** | 모든 공유 데이터(사망 플래그, 식사 시간)에 뮤텍스 적용 |
| **정확한 타이밍** | `gettimeofday()`로 밀리초 단위 정밀 측정 |

### 구조

- **메인 스레드:** 각 철학자를 위한 스레드 생성 + 사망 감시
- **철학자 스레드:** 생각/포크 획득/식사/수면 루프 반복
- **포크:** 각각 별도의 뮤텍스로 보호

---

## 배운 점

- `pthread_create` / `pthread_join` 스레드 생명주기 관리
- `pthread_mutex_lock` / `pthread_mutex_unlock` 으로 임계 구역(critical section) 보호
- 데드락의 원인과 포크 획득 순서를 이용한 방지 기법
- 경쟁 조건(Race Condition)을 검출하고 방지하는 설계 패턴
- `usleep` 기반 타이밍 제어와 정밀도의 트레이드오프

# minishell

> 42 Gyeongsan — Circle 3

Bash 셸을 직접 재구현한 프로젝트입니다.  
  
토크나이저부터 실행기까지 전 파이프라인을 구현했습니다.

---

## 파이프라인 구조

```
사용자 입력
    │
    ▼
[Tokenizer]  ── 입력 문자열을 토큰 단위로 분리
    │              (단어, 파이프, 리디렉션, 문자열 등)
    ▼
[Parser]     ── 토큰을 명령어 구조체로 파싱
    │              (커맨드, 인자, 리디렉션 정보)
    ▼
[Executor]   ── 실행
               ├─ 빌트인이면 직접 실행
               └─ 외부 명령어: fork → execve
                   ├─ 파이프라인: 여러 자식 프로세스 + pipe()
                   └─ 리디렉션: dup2()로 fd 교체
```

---

## 구현 기능

### 핵심 기능
- **파이프라인** (`cmd1 | cmd2 | cmd3`)
- **리디렉션** (`>`, `<`, `>>`, `<<` heredoc)
- **환경변수 확장** (`$VAR`, `$?`)
- **시그널 처리** (`Ctrl+C`, `Ctrl+D`, `Ctrl+\`)
- **따옴표 처리** (큰따옴표 내 변수 확장, 작은따옴표 리터럴)

### 빌트인 명령어

| 명령어 | 기능 |
|--------|------|
| `echo` | `-n` 플래그 포함 |
| `cd` | 상대/절대 경로, `~` 지원 |
| `pwd` | 현재 경로 출력 |
| `export` | 환경변수 설정 및 목록 출력 |
| `unset` | 환경변수 제거 |
| `env` | 환경변수 전체 출력 |
| `exit` | 종료 코드와 함께 셸 종료 |

## 소스 구조

| 파일 | 역할 |
|------|------|
| `tokenizer*.c` | 문자열 → 토큰 분리 |
| `parser*.c` | 토큰 → 커맨드 구조체 |
| `execute*.c` | 명령어 실행, 파이프, 리디렉션 |
| `builtin*.c` | 빌트인 명령어 구현 |
| `env*.c` | 환경변수 관리 |
| `signal.c` | 시그널 핸들러 등록 및 처리 |

## 실행 방법

```bash
make
./minishell

$ echo "Hello, World!"
$ ls -la | grep .c | wc -l
$ cat < input.txt >> output.txt
$ export MY_VAR=42
$ cd /tmp && pwd
```

## 배운 점

- `fork()` / `execve()` / `waitpid()` 를 이용한 프로세스 생성과 관리
- `pipe()` / `dup2()` 로 파이프라인 구현
- 렉서-파서 구조로 문법 파싱 아키텍처 설계
- 셸의 인용 규칙(quoting rules)과 변수 확장 우선순위
- 시그널을 적절한 컨텍스트(prompt vs. 실행 중)에서 다르게 처리
- Valgrind로 메모리 누수를 추적하고 수정하는 디버깅 실전 경험

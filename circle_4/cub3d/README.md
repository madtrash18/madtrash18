# cub3d

> 42 Gyeongsan — Circle 4

**레이캐스팅(Raycasting)** 기법을 이용해 2D 맵을 3D 시점으로 렌더링하는 그래픽 엔진입니다.  
Wolfenstein 3D와 같은 고전 FPS 게임의 렌더링 원리를 직접 수학적으로 구현했습니다.

---

## 동작 원리

```
플레이어 위치에서 화면 너비만큼(열마다) 광선을 발사합니다.

각 광선:
  ├─ DDA 알고리즘으로 벽에 닿을 때까지 격자 이동
  ├─ 벽까지의 거리 계산
  ├─ 거리에 반비례하는 높이로 벽 슬라이스 렌더링
  └─ 벽 방향(N/S/E/W)에 따라 해당 텍스처 적용
```

### DDA (Digital Differential Analysis)

- 광선 방향 벡터와 격자 교점 간 거리를 이용해 효율적으로 벽 탐색
- 수평/수직 충돌 중 가까운 것을 선택

---

## 구현 기능

| 기능 | 상세 |
|------|------|
| **3D 렌더링** | DDA 레이캐스팅, 원근 투영 |
| **텍스처 매핑** | 벽 방향별 (North/South/East/West) XPM 텍스처 |
| **플레이어 이동** | 전/후/좌/우 이동, 좌/우 회전 |
| **맵 파싱** | `.cub` 파일에서 맵·설정 파싱, 유효성 검증 |
| **색상 설정** | 바닥/천장 RGB 색상 지정 |

---

## `.cub` 파일 형식

```
NO ./textures/north.xpm
SO ./textures/south.xpm
WE ./textures/west.xpm
EA ./textures/east.xpm

F 220,100,0
C 225,30,0

111111
100001
1N0001
100001
111111
```

---

## 실행 방법

```bash
make
./cub3D example.cub
```

**조작:**
- `W`/`A`/`S`/`D` — 이동
- `←` / `→` — 시점 회전
- `ESC` — 종료

---

## 소스 구조

| 파일 | 역할 |
|------|------|
| `main.c` | 진입점, 초기화 |
| `parser.c~3.c` | `.cub` 파일 파싱 및 유효성 검사 |
| `raycaster.c~2.c` | DDA 레이캐스팅, 렌더링 |
| `event.c` | 키 입력 처리 |
| `init.c` | MiniLibX 윈도우/이미지 초기화 |
| `cleanup.c` | 메모리 해제 |

---

## 배운 점

- 레이캐스팅과 원근 투영의 수학적 원리 (벡터, 삼각함수)
- DDA 알고리즘의 효율성과 격자 기반 충돌 탐지
- MiniLibX의 이중 버퍼링(mlx_image)으로 화면 깜빡임 제거
- 복잡한 렌더링 루프에서의 성능 최적화
- 텍스처 샘플링: UV 좌표 계산과 픽셀 매핑

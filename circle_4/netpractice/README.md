# NetPractice

> 42 Gyeongsan — Circle 4

TCP/IP 네트워킹의 기초를 실습으로 익히는 이론·실습 과제입니다.  
10개의 레벨로 구성된 네트워크 문제를 직접 풀어가며 서브네팅과 라우팅을 학습합니다.

---

## 학습 내용

### IP 주소와 서브넷 마스크

| 개념 | 내용 |
|------|------|
| **IPv4 구조** | 32비트 주소, 네트워크 + 호스트 부분 |
| **서브넷 마스크** | CIDR 표기법 (`/24` = `255.255.255.0`) |
| **네트워크 주소** | AND 연산으로 계산 |
| **브로드캐스트** | 호스트 비트 전부 1 |
| **사용 가능 호스트** | 전체 - 2 (네트워크, 브로드캐스트 제외) |

### 라우팅

- **기본 게이트웨이:** 같은 네트워크가 아닌 목적지로 가는 기본 경로
- **정적 라우팅 테이블:** 목적지 IP + 넥스트 홉 IP 설정
- **루프백:** `127.0.0.1/8` 자기 자신

---

## 실습 레벨 구성

| 레벨 | 주요 개념 |
|------|----------|
| 1~3 | 기본 IP 설정, 같은 서브넷 내 통신 |
| 4~6 | 다중 인터페이스, 라우터 구성 |
| 7~8 | 서브네팅, 여러 네트워크 분리 |
| 9~10 | 복잡한 멀티 라우터 토폴로지 |

---

## 핵심 공식

```
네트워크 주소   = IP AND 서브넷마스크
브로드캐스트    = 네트워크 주소 OR (NOT 서브넷마스크)
사용 가능 범위  = 네트워크 주소 + 1  ~  브로드캐스트 - 1

예시: 192.168.1.0/25
  → 서브넷마스크: 255.255.255.128
  → 사용 가능: 192.168.1.1 ~ 192.168.1.126
  → 호스트 수: 126개
```

## 배운 점

- TCP/IP 계층 모델과 네트워크 주소 계산 원리
- CIDR 표기법과 서브넷 마스크를 이용한 네트워크 분할
- 라우팅 테이블 설계 및 경로 선택 규칙
- 실제 네트워크 토폴로지를 다이어그램으로 해석하는 능력

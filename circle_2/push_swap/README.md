# push_swap

> 42 Gyeongsan — Circle 2

두 개의 스택(`a`, `b`)과 제한된 연산만을 사용하여 정수 배열을 **최소한의 명령어 수**로 정렬하는 알고리즘 프로젝트입니다.

---

## 허용 연산

| 명령어 | 설명 |
|--------|------|
| `sa` / `sb` / `ss` | 스택의 최상단 두 요소 교환 |
| `pa` / `pb` | 반대 스택의 top을 현재 스택으로 이동 |
| `ra` / `rb` / `rr` | 스택을 위로 회전 (top → bottom) |
| `rra` / `rrb` / `rrr` | 스택을 아래로 회전 (bottom → top) |

---

## 알고리즘

### 소규모 (≤ 5개)
- 하드코딩된 최적 패턴으로 처리

### 중·대규모
**인덱싱 + Greedy 전략:**

1. **정규화(Indexing):** 값을 순위(index)로 치환해 비교 비용 감소
2. **청크 분할:** `a`의 요소를 크기 범위로 나눠 순서대로 `b`로 이동
3. **Greedy 삽입:** `b`의 각 요소를 `a`의 올바른 위치에 가장 적은 연산으로 삽입
   - `a`의 회전 비용 + `b`의 회전 비용을 합산해 최소 비용 위치 선택
4. 최종 정렬 후 최솟값을 top으로 회전

| 측정 항목 | 결과 |
|-----------|------|
| 최대 연산 수 (500개) | ~5500 이하 |

## 실행 방법

```bash
make

ARG="3 1 4 1 5"; ./push_swap $ARG | ./checker $ARG
# → OK
```

## 배운 점

- 알고리즘 최적화: 동일한 결과를 더 적은 연산으로 달성하는 설계
- Greedy 알고리즘의 실전 적용과 한계 이해
- 덱(Deque) 자료구조를 이용한 효율적인 스택 시뮬레이션
- 정규화(normalization)로 비교 복잡도를 줄이는 기법

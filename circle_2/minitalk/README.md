# minitalk

> 42 Gyeongsan — Circle 2

UNIX 시그널(`SIGUSR1`, `SIGUSR2`)만을 사용하여 클라이언트에서 서버로 문자열 메시지를 전송하는 IPC(프로세스 간 통신) 프로그램입니다.

---

## 동작 원리

문자의 각 비트를 시그널로 인코딩하여 전송합니다.

```
클라이언트                          서버
  │                                  │
  │  각 문자의 비트를 순서대로 전송    │
  │  ─────────────────────────────▶  │
  │  bit=1 → SIGUSR1 전송            │  시그널 핸들러에서 비트 수신
  │  bit=0 → SIGUSR2 전송            │  8비트 모이면 문자 완성
  │                                  │  → putchar 출력
```

### 흐름

1. **서버** 실행 → 자신의 PID 출력
2. **클라이언트** `./client [서버 PID] [메시지]` 실행
3. 메시지의 각 문자를 8비트로 분해하여 시그널로 전송
4. 서버는 시그널을 수신하며 비트를 재조합 → 문자 출력

---

## 실행 방법

```bash
make

# 터미널 1 - 서버 실행
./server
# → PID: 12345

# 터미널 2 - 클라이언트 실행
./client 12345 "Hello, 42!"
```

---

## 보너스

- 서버 → 클라이언트 수신 확인 시그널 전송 (acknowledgment)
- 유니코드 문자 지원

---

## 배운 점

- `signal()` / `sigaction()` 의 차이와 안전한 핸들러 작성
- 비트 연산 (`<<`, `>>`, `&`, `|`) 을 통한 데이터 인코딩/디코딩
- 프로세스 ID(PID)와 `kill()` 함수로 프로세스 간 통신
- `usleep()`을 이용한 타이밍 제어 (시그널 손실 방지)
- `volatile sig_atomic_t`를 이용한 시그널 핸들러 안전 변수
